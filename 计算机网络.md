*归零编码*

每个时钟周期的中间跳转为0

*非归零编码*

不用归零

*反向非归零编码*

翻转代表0，不变代表1

*曼彻斯特编码*

将码元分为两个相等的间隔，前一个间隔电平为高后一个电平为低代表1，0的表示方法相反

*差分曼彻斯特编码*

翻转代表0，不变代表1

慢开始：cwnd<阈值ssthresh时，每经过一个RTT，cwnd加倍（指数增长）

拥塞避免：cwnd>阈值ssthresh时，每经过一个RTT，cwnd+1（线性增长）

缓解拥塞：令阈值ssthresh=cwnd/2，cwnd=1

*快重传和快恢复*![img](https://uploader.shimo.im/f/wAATWc77zoAnAIpd.png!thumbnail?accessToken=eyJhbGciOiJIUzI1NiIsImtpZCI6ImRlZmF1bHQiLCJ0eXAiOiJKV1QifQ.eyJleHAiOjE3NTk4NDQyMTgsImZpbGVHVUlEIjoiMndBbGQ1TTUxbGZtcHBBUCIsImlhdCI6MTc1OTg0MzkxOCwiaXNzIjoidXBsb2FkZXJfYWNjZXNzX3Jlc291cmNlIiwicGFhIjoiYWxsOmFsbDoiLCJ1c2VySWQiOjUxMTEyODEzfQ.lPb7gje7G6xbOkEpt7OK43O4i0vQX9W2UJpdR3lXkVo)

采用C/S模型，运行在TCP上，控制连接为21号端口，数据连接为20号端口![img](https://uploader.shimo.im/f/KTBf7t0ka5mNxRPI.png!thumbnail?accessToken=eyJhbGciOiJIUzI1NiIsImtpZCI6ImRlZmF1bHQiLCJ0eXAiOiJKV1QifQ.eyJleHAiOjE3NTk4NDQyMTgsImZpbGVHVUlEIjoiMndBbGQ1TTUxbGZtcHBBUCIsImlhdCI6MTc1OTg0MzkxOCwiaXNzIjoidXBsb2FkZXJfYWNjZXNzX3Jlc291cmNlIiwicGFhIjoiYWxsOmFsbDoiLCJ1c2VySWQiOjUxMTEyODEzfQ.lPb7gje7G6xbOkEpt7OK43O4i0vQX9W2UJpdR3lXkVo)

主机A想下载文件ftp://ftp.abc.edu.cn/file，大致描述过程：

1、主机A建立一个TCP连接到服务器ftp.abc.edu.cn的21号端口，建立控制连接，发送登陆账号和密码

2、服务器返回登陆成功信息，主机A打开一个随机端口，并将端口发发送给服务器

3、主机A发送读文件命令，内容为get file，服务器使用20号端口建立一个TCP连接到主机A随机打开的端口，建立数据连接

4、服务器把内容通过数据连接发送给主机A，传输完毕后连接关闭

***HTTP——超文本传输协议\***

*定义*

![img](https://uploader.shimo.im/f/0KOJ4MClGR9a4KDy.png!thumbnail?accessToken=eyJhbGciOiJIUzI1NiIsImtpZCI6ImRlZmF1bHQiLCJ0eXAiOiJKV1QifQ.eyJleHAiOjE3NTk4NDQyMTgsImZpbGVHVUlEIjoiMndBbGQ1TTUxbGZtcHBBUCIsImlhdCI6MTc1OTg0MzkxOCwiaXNzIjoidXBsb2FkZXJfYWNjZXNzX3Jlc291cmNlIiwicGFhIjoiYWxsOmFsbDoiLCJ1c2VySWQiOjUxMTEyODEzfQ.lPb7gje7G6xbOkEpt7OK43O4i0vQX9W2UJpdR3lXkVo)

*工作过程*

1、浏览器分析链接指向页面的URL，向DNS请求解析网站的IP地址

2、DNS解析出IP地址，浏览器与服务器建立TCP连接，浏览器发出HTTP请求

3、服务器通过HTTP响应把文件发给浏览器，释放TCP连接，浏览器解释文件并把web页面显示给用户

*报文结构*

![img](https://uploader.shimo.im/f/kM70cw4SyjknbGCu.png!thumbnail?accessToken=eyJhbGciOiJIUzI1NiIsImtpZCI6ImRlZmF1bHQiLCJ0eXAiOiJKV1QifQ.eyJleHAiOjE3NTk4NDQyMTgsImZpbGVHVUlEIjoiMndBbGQ1TTUxbGZtcHBBUCIsImlhdCI6MTc1OTg0MzkxOCwiaXNzIjoidXBsb2FkZXJfYWNjZXNzX3Jlc291cmNlIiwicGFhIjoiYWxsOmFsbDoiLCJ1c2VySWQiOjUxMTEyODEzfQ.lPb7gje7G6xbOkEpt7OK43O4i0vQX9W2UJpdR3lXkVo)

开始行：请求报文中称为“请求行”，响应报文中称为“状态行”

请求报文的方法：

| 方法（操作） | 意义                          |
| ------------ | ----------------------------- |
| GET          | 请求读取由URL标识的信息       |
| HEAD         | 请求读取由URL标识的信息的首部 |
| POST         | 给服务器添加信息（如注释）    |
| CONNECT      | 用于代理服务器                |
